# OCR 文字识别 Skill

## 功能说明

你可以识别图片中的文字，支持：
- 中文（简体/繁体）
- 英文
- 日语、韩语等多国语言

## 使用流程

当用户运行 `/ocr <图片路径>` 命令时：

### 第一步：验证图片文件
1. 检查用户提供的图片路径是否存在
2. 验证文件格式是否支持（png, jpg, jpeg, bmp, tiff 等）
3. 如果文件不存在或格式不支持，提示用户并提供示例

### 第二步：分析图片内容（可选）
如果用户不确定 OCR 参数，可以先用 MCP 图像分析工具预览图片：
- 使用 `mcp__zai-mcp-server__analyze_image` 工具查看图片内容
- 判断图片主要语言（中文/英文/混合）
- 评估文字清晰度和复杂度

### 第三步：转换为 Base64
1. 使用 Python 将图片转换为 base64 编码：
   ```python
   import base64
   with open('图片路径', 'rb') as f:
       base64_data = base64.b64encode(f.read()).decode()
   ```
2. 不要直接输出 base64（太大），而是保存到变量中

### 第四步：确定 OCR 参数

根据图片内容选择合适的参数：

**语言选择（lang）：**
- `chi_sim+eng` - 中英文混合（默认推荐）
- `eng` - 纯英文
- `chi_sim` - 纯中文简体
- `chi_tra` - 中文繁体
- `jpn` - 日语
- `kor` - 韩语
- 支持多种语言组合，用 `+` 分隔

**预处理（preprocess）：**
- `true` - 启用预处理（默认推荐，可提高识别率）
  - 包括：灰度化、锐化、自动对比度增强
- `false` - 不预处理（图片质量很好时可选择）

**页面分割模式（psm）：**
- `6` - 假设文本为单一统一块（默认推荐）
- `3` - 全自动页面分割
- `7` - 将图像视为单行文本
- `13` - 原始行（保留任意间距）

### 第五步：调用 MCP OCR API

使用 `mcp__ocr-service__ocr_recognize_base64` 工具：

```json
{
  "file_name": "图片文件名",
  "image_base64": "<base64编码的图片数据>",
  "lang": "chi_sim+eng",
  "preprocess": true,
  "psm": 6
}
```

### 第六步：返回识别结果

1. **成功时：**
   - 显示识别出的文字（text 字段）
   - 如果有多行，显示行数统计
   - 如果有特定格式（如表格、列表），优化展示

2. **失败时：**
   - 显示错误信息
   - 提供可能的原因和解决方案
   - 例如：图片格式不支持、base64 编码错误、网络问题等

## 高级技巧

### 处理大文件
如果图片很大（> 2MB）：
1. 建议用户先压缩图片
2. 或使用图片编辑工具裁剪到需要的区域
3. MCP OCR 支持 max 8MB 的文件

### 处理倾斜图片
如果图片文字倾斜：
- 建议用户先旋转校正图片
- 或者尝试不同的 psm 参数（如 13 - 原始行模式）

### 提高识别率
如果识别结果不准确：
1. 启用预处理（preprocess=true）
2. 确保图片清晰度足够（建议 > 300 DPI）
3. 调整对比度和亮度
4. 尝试不同的语言模式
5. 对于表格，建议使用专业表格 OCR 工具

## MCP 工具参考

可用的 MCP OCR 工具：
- `mcp__ocr-service__ocr_recognize_base64` - 通过 base64 识别（推荐）
- `mcp__ocr-service__ocr_recognize_file` - 通过服务器文件路径识别
- `mcp__ocr-service__ocr_health` - 检查 OCR 服务健康状态

## 注意事项

1. **隐私安全**：图片会通过 HTTP 上传到远程 OCR 服务器（http://api.cstwy.top），请勿上传敏感信息
2. **性能考虑**：base64 转换和 OCR 识别都需要时间，大文件可能需要 10-30 秒
3. **并发限制**：OCR 服务有并发限制，队列等待超时时间为 4 秒
4. **超时设置**：单次 OCR 操作超时时间为 25 秒

## 故障排查

**问题 1：Base64 数据太大无法传递**
- 解决：图片文件 > 500KB 时，使用临时文件或流式处理

**问题 2：OCR 返回 400 错误**
- 检查 base64 编码是否正确
- 验证图片格式是否支持
- 尝试不同的 psm 参数

**问题 3：识别结果为空**
- 检查图片是否包含文字
- 尝试启用预处理
- 调整语言参数

**问题 4：识别准确率低**
- 提高图片清晰度
- 裁剪到只包含文字的区域
- 尝试不同的预处理选项
